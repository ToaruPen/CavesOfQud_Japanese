# テストプラン (スモーク)

| シナリオ | 手順 | 観点 |
| --- | --- | --- |
| メインメニュー | ゲーム起動→タイトル→設定画面 | ロゴ/メニューのフォント・折返し・ボタンラベル |
| キャラクリ | New Game→キャラクター作成→背景/職業選択 | 長文説明とツールチップの折返し、選択肢幅 |
| 会話イベント | ヨッパ（Joppa）/Argyve など主要 NPC に話しかける | 会話ログの禁則・色タグ・選択肢番号 |
| 書籍閲覧 | 図書館/ベッドで入手できる書籍を読む | 書籍ビューアの段落間隔、スクロール挙動 |
| 戦闘ログ | 砂漠で複数エネミーと戦闘 | 動的テキスト（命中/回避/ステータス）と色タグ |
| インベントリ | 収集品 + トレード画面 | 多列レイアウトでの行揃え/溢れ判定 |
| ジャーナル | アチーブメント/任務画面 | 自動生成文 + UI の改行 |

## テストフロー
1. テスト前に `build_log.txt` と `Player.log` をクリア。
2. シナリオを実行し、表示崩れ・誤訳・抜け文字を確認。
3. 問題があればスクリーンショット + ログ抜粋を Issue または Docs/test_report.md へ記録。

## Pass 条件
- 致命的な UI 閉塞 (文字化けで進行不可) がゼロ。
- Player.log に MissingGlyph / Harmony Exception が残っていない。
- 翻訳済みセクションで英語文が混在しない。
